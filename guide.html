User Guide

Everything you need to know to use, update, and maintain the Mission 300 Clean Cooking website.

Section 1

Overview

The Mission 300 website is a single-page data visualization site for the Sustainable Energy for All (SEforALL) initiative. It presents clean cooking analysis data for 30 African countries sourced from National Energy Compact documents.

Key Features

  • Interactive country profiles with flags, multi-dimensional filters, and search
  • Gap analysis chart — current access vs. 2030 targets
  • Investment breakdown — public vs. private financing
  • Barrier and technology pathway analysis cards
  • Detailed country modal with full statistics
  • Responsive design — desktop, tablet, and mobile

Tech Stack

LayerTechnology
FrontendVanilla HTML / CSS / JavaScript (no framework)
FontsGoogle Fonts — Barlow Condensed + Mulish
Flagsflagcdn.com CDN
HostingGitHub Pages
No build tools are required. Edit the files directly and push to GitHub to deploy.
Section 2

Using the Website

A guide for end-users navigating the live site.

Navigation

  • Use the top navigation bar to jump to any section: Overview, Countries, Data, Investment, Challenges, Technology
  • On mobile, tap the hamburger menu (three lines) to open navigation

Filtering Countries

The Countries section has three filter groups that combine with AND logic:

FilterOptions
CohortAll (30), Cohort 1 (12), Cohort 2 (18)
RegionAll, West Africa (11), East Africa (8), Central Africa (6), Southern Africa (5)
LanguageAll, French (15), English (15)

Click any filter button (turns navy blue when active). A summary shows: "Showing X countries matching: [filters]". Use the search bar to further filter by name.

Viewing Country Details

Click any country card to open a modal with full statistics, technology focus, strategy status, and key barriers. Close with the X button, clicking outside, or pressing Escape.

Reading Charts

  • Gap Analysis: Yellow bars = current access, faded green = 2030 target. The gap shows how far each country needs to go.
  • Investment: Navy = public finance, green = private finance. Bar length shows relative amounts.

Both are defined in data.js and rendered automatically.

data.js
{
  title: "Hydrogen Fuel Cells",
  description: "Emerging hydrogen technology",

Country names in the countries array must exactly match the country field in the countries data for flag lookup to work.

Section 8

Change Branding

Colors

All colors are CSS custom properties in styles.css under :root. Change the hex value to update everywhere:

styles.css
--yellow: #FFB800;       /* Primary accent */
--navy: #065A89;         /* Secondary brand */
--dark-navy: #022437;    /* Text & backgrounds */
--green: #9BBE1D;        /* Positive data */
--red: #DD2E2B;          /* Negative data */
Some colors are also hardcoded in index.html SVG elements (hero ring strokes and logo). Update those too.

Fonts

  1. Pick new fonts at fonts.google.com
  2. Replace the <link> tag in index.html
  3. Update --font-sans and --font-display in styles.css

Add a New Filter

Example: adding an "Investment Range" filter.

Step 1 — Add data attribute in app.js

app.js — renderCountryCards()
data-investment-range="${c.totalInvestment > 500 ? 'high' : 'low'}"

Step 2 — Add filter buttons in index.html

index.html
<div class="filter-group">
  <span class="filter-group-label">Investment</span>
  <div class="filter-bar">
    <button class="filter-btn active"
      data-filter="all" data-type="investmentRange">All</button>
    <button class="filter-btn"
      data-filter="high" data-type="investmentRange">High</button>
  </div>
</div>

Step 3 — Update filter state in app.js

app.js
// Add to activeFilters object:
investmentRange: 'all'

// Add to applyFilters():
const matchesInvestment = activeFilters.investmentRange === 'all' ||
  card.dataset.investmentRange === activeFilters.investmentRange;

Section 12

Troubleshooting

Click any issue to expand the solution.

Hard refresh with Ctrl+Shift+R (Cmd+Shift+R on Mac). Verify the iso code in data.js is valid by visiting https://flagcdn.com/w40/{iso}.png.

Check for: missing commas between objects, correct region/language/cohort values matching filter buttons exactly. Open browser DevTools (F12) > Console for JS errors.

Filter counts are hardcoded in index.html, not calculated. After adding/removing countries, update the <span class="filter-count"> values manually.

Bar widths are relative to the max value. Adding a country with much higher investment will make other bars smaller (expected). The access chart filters out countries with target2030: null.

Filenames are case-sensitive on Linux. Make sure names match exactly. Run git add photos/ && git commit -m "add photos" && git push. GitHub Pages has a ~1GB soft limit; compress large images.

GitHub Pages takes 1-2 minutes to rebuild. Check deployment status at the repo's Actions tab. Hard refresh your browser to bypass cache.
